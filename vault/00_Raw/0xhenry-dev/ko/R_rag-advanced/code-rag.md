### 서론

코드베이스에 RAG를 적용하는 것은 일반 에세이 모음집에 적용하는 것과 근본적으로 다릅니다. 일반 문서의 문맥은 주로 선형적(문단 내의 문장들)입니다. 반면, 코가의 문맥은 **관계적이고 구조적**입니다. `auth.py`에 있는 함수가 `utils.py`에 정의된 클래스에 의존할 수 있죠. 만약 이 파일들을 단순히 글자 수 단위로 무작위로 자르면, AI는 코드 간의 연결 고리를 완전히 잃어버리게 됩니다.

진정한 **코드 RAG(Code RAG)**를 구축하려면 '코드 구조를 인식하는' 분할 방식이 필요합니다.

---

### 코드 문맥의 난제들

1.  **임포트(Import)와 의존성**: 파일을 이해하기 위해 AI는 해당 파일이 무엇을 임포트하는지 알아야 합니다.
2.  **클래스 계층 구조**: 현재 파일의 메서드가 비어 있더라도, 다른 곳에 정의된 부모 클래스에 실제 로직이 있을 수 있습니다.
3.  **비선형적 흐름**: 코드 실행은 파일과 파일 사이를 수시로 넘나듭니다.

### 해결책: 구조적 인덱싱 (AST 활용)

단순히 글자 수로 자르는 대신, **추상 구문 트리(AST, Abstract Syntax Tree)** 파서를 사용하여 논리적 단위로 분할합니다:

-   **함수**: 각 함수를 독립적인 청크로 만듭니다.
-   **클래스**: 클래스 정의와 메타데이터를 하나의 청크로 묶습니다.
-   **요약(Summary)**: 각 파일에 대해 고수준의 요약 정보(예: "이 파일은 JWT 토큰 검증을 담당함")를 생성하여, 전반적인 아키텍처 질문에 답할 수 있게 합니다.

---

### 코드 RAG를 위한 도구들

-   **Tree-sitter**: 거의 모든 프로그래밍 언어의 구문 트리를 빌드할 수 있는 라이브러리입니다. 함수와 클래스를 수술용 칼로 도려내듯 정밀하게 추출할 수 있습니다.
-   **Greptile / Bloop**: 코드베이스 인덱싱에 특화된 엔터프라이즈급 서비스들입니다.
-   **LangChain CodeSplitter**: Python, JS, Go 등의 기본 구문을 이해하여 일반 텍스트 분할기보다 훨씬 깔끔하게 코드를 나눠주는 도구입니다.

---

### 구현 전략

1.  **태깅(Tagging)**: 모든 청크에 파일 경로, 라인 번호, 함수 이름 등의 메타데이터를 추가합니다.
2.  **의존성 매핑**: 각 청크에 해당 코드가 사용하는 임포트 정보의 '시그니처'를 포함시켜, AI가 관련 로직을 어디서 찾아야 할지 알게 합니다.
3.  **'Map-Reduce' 접근법**: "로그인 흐름이 어떻게 돼?" 같은 광범위한 질문을 받으면, 먼저 파일 요약들을 검색하고, 그 요약에서 언급된 특정 함수들을 다시 검색하는 단계를 거칩니다.

---

### 요약

코드 RAG는 개발자 온보딩과 디버깅 속도를 비약적으로 높여주는 궁극의 도구입니다. '텍스트 검색'에서 '구문 인식 검색'으로 진화함으로써, 여러분의 AI는 단순한 문서 리더기를 넘어 여러분의 아키텍처를 진정으로 이해하는 코파일럿(Co-pilot)이 될 것입니다.

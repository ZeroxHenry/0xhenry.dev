# 🌉 이미지 협업 브릿지 사용 가이드

에이전트의 이미지 생성 쿼터가 부족할 때, 사용자가 생성한 이미지를 자동으로 처리하여 블로그 포스트에 연결하는 도구입니다.

## 🛠 사용 방법

1. **상태 확인**: `tasks.json` 파일을 열어 `pending` 상태인 프롬프트 목록을 확인합니다.
2. **이미지 생성**: 프롬프트를 복사하여 제미나이(또는 다른 도구)로 이미지를 생성합니다.
3. **파일 넣기**: 생성된 이미지 파일들을 이 폴더의 `input/` 디렉토리에 넣습니다.
   - *팁: 파일 이름은 상관없습니다. 에이전트가 `tasks.json`의 순서대로 가져가서 이름을 바꿉니다.*
4. **처리 실행**: 에이전트에게 "이미지 브릿지 처리해줘"라고 말하거나, 직접 스크립트를 실행합니다:
   ```bash
   python3 scripts/process_user_images.py
   ```

## ✨ 에이전트가 하는 일
- 이미지 하단의 **Gemini 워터마크 자동 제거** (하단 60px 크롭)
- 포스트 ID에 맞게 **파일 이름 변경** 및 `images/2026-04/` 폴더로 이동
- `post.md` 본문에 이미지 경로가 올바르게 삽입되어 있는지 확인 (이미 되어 있음)
- `tasks.json` 상태를 `completed`로 변경

## ⚠️ 주의사항
- `Pillow` 라이브러리가 필요합니다: `python3 -m pip install Pillow`
- 이미지 비율은 가급적 **16:9**를 권장합니다 (네이버 블로그 최적화).

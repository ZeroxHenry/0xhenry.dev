사용자: "우리 회사의 올해 하반기 복지 포인트 정책이 뭐야?"
AI: (데이터에 없는데도) "네, 올해 하반기 복지 포인트는 200만 원이며, 10월부터 지급될 예정입니다!"

실제 정책은 아직 확정도 안 됐는데 AI가 당당하게 거짓말을 합니다. 이것이 바로 **확신에 찬 할루시네이션(Confident Hallucination)**입니다. 

RAG(Retrieval-Augmented Generation) 시스템에서 가장 어려운 것은 AI가 답하게 만드는 것이 아니라, **답하지 말아야 할 때를 알게 하는 것**입니다.

---

### 왜 AI는 '모른다'고 안 할까?

LLM의 본능은 **'다음 단어를 예측하는 것'**입니다. 프롬프트에 아무리 "모르면 모른다고 해"라고 써도, 배경 지식(Parametric Knowledge)과 검색된 문서(Context) 사이의 경계가 흐려지면 AI는 어떻게든 창의력을 발휘해 빈칸을 채우려 합니다.

이것을 차단하기 위한 3단계 트리거 시스템을 구축해야 합니다.

---

### 1단계: 검색 품질 점수(Relevance Score) 컷오프

가장 원초적이면서 확실한 방법입니다. 리트리버가 문서를 가져왔을 때, 유사도 점수(Cosine Similarity 등)가 특정 임계값(Threshold) 이하라면 모델에게 "문서가 없으니 답하지 마"라고 미리 알려주는 것이죠.

```python
def retrieve_with_safety(query, index, threshold=0.75):
    results = index.search(query, top_k=3)
    
    # 가장 높은 점수가 임계값보다 낮으면 '데이터 없음' 처리
    if results[0].score < threshold:
        return "NO_CONTEXT_FOUND"
    
    return results
```

---

### 2단계: 네거티브 프롬프트(Negative Prompt) 강화

단순히 "모르면 모른다고 해"라고 하면 부족합니다. **"제공된 문서에 답이 절대적으로 없다면 다음과 같이 답하세요: '죄송합니다. 제공된 문서에서 해당 정보를 찾을 수 없습니다.'"**라고 매우 구체적인 출력 형식을 지정해야 합니다.

또한, **"당신의 사전 지식을 절대 사용하지 마세요"**라는 명시적 금지 명령이 포함되어야 합니다.

---

### 3단계: 근거 기반 검증 (NLI - Natural Language Inference)

생성된 답변이 실제 문서를 '함의(Entailment)'하고 있는지 별도의 가벼운 모델로 검사하는 기법입니다.

- **전제(Premise)**: 검색된 문서 내용
- **가설(Hypothesis)**: AI가 생성한 답변
- **검증**: 가설이 전제로부터 논리적으로 도출되는가? (Neutral이나 Contradiction이면 필터링)

이 과정을 통해 AI가 문서를 '참고'만 한 것인지, 아니면 문서를 넘어서 '창작'을 한 것인지 판별할 수 있습니다.

---

### Henry의 팁: "모른다"는 답변 뒤에 무엇을 할 것인가?

단순히 "모릅니다"라고 끝내는 것은 나쁜 UX입니다. 신뢰성 있는 AI는 다음과 같이 제안해야 합니다.

1. **대안 제시**: "그 정보는 없지만, 관련된 '작년 복지 정책'에 대해 알려드릴까요?"
2. **출처 확인 요청**: "제가 가진 문서 중 '복지' 카테고리에는 해당 내용이 없습니다. 문서를 추가해 주시면 다시 확인해 보겠습니다."
3. **피드백 루프**: 사용자가 "왜 몰라?"라고 했을 때, 어떤 문서를 찾았었는지 투명하게 보여주는 것이 신뢰의 핵심입니다.

---

### 결론

확신에 찬 거짓말은 AI 시스템에 대한 신뢰를 한순간에 무너뜨립니다. **"모른다"고 말하는 용기**를 에이전트에게 심어주는 것이야말로 프러덕션용 AI의 완성입니다.

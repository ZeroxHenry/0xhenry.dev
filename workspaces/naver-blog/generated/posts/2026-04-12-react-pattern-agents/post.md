# ReAct 패턴: 에이전트가 생각하고 행동하는 방식

---

### 서론

AI는 언제 웹 검색을 해야 할지, 언제 계산기를 사용해야 할지, 그리고 언제 마침내 정답을 찾았는지를 어떻게 알까요? 에이전트 도입 초기에는 LLM이 도구 사용법을 마음대로 지어내는 '할루시네이션' 현상이 잦았습니다. 이를 획기적으로 해결한 돌파구가 바로 **ReAct**(Reason + Act) 패턴입니다.

ReAct는 LLM이 어떤 행동을 취하기 전에 반드시 자신의 내부 생각 과정을 글로 남기도록 강제하는 프롬프팅 기술입니다.

---

### ReAct 루프: Thought, Action, Observation

ReAct 패턴은 다음 세 단계를 반복하는 루프로 구성됩니다:

1.  **Thought (생각)**: LLM이 현재 상황에서 무엇을 해야 할지 생각합니다.
    -   *예*: "5 비트코인(BTC)의 총 가치를 계산하기 위해 먼저 현재 비트코인 가격을 찾아봐야겠다."
2.  **Action (행동)**: LLM이 도구를 선택하고 입력값을 제공합니다.
    -   *예*: `Search(query="현재 비트코인 시세")`
3.  **Observation (관찰)**: 시스템이 도구를 실행하고 그 결과를 LLM에게 다시 전달합니다.
    -   *예*: "현재 비트코인 가격은 1억 3천만 원입니다."

LLM은 이 **Observation**을 보고 다시 새로운 **Thought**와 함께 루프를 시작합니다. ("가격이 1억 3천만 원임을 확인했으니, 이제 여기에 5를 곱하겠다...")

---

### 왜 ReAct가 중요한가요?

-   **설명 가능성 (Explainability)**: "Thought" 섹션을 읽어보면 AI가 왜 특정 도구를 선택했는지 정확히 알 수 있습니다. 실수가 발생했을 때 어느 논리 단계에서 어긋났는지 파악하기 쉽습니다.
-   **오류 수정 (Error Correction)**: 도구가 오류(예: "페이지를 찾을 수 없음")를 반환하면, LLM은 Observation에서 이를 확인하고 정보를 얻기 위한 다른 방법을 '생각'할 수 있습니다.
-   **시너지**: '생각의 사슬(Chain-of-Thought)' 추론 능력과 외부 도구 활용 능력을 하나로 결합합니다.

---

### 구현 예시 (개념도)

LangChain에서는 이를 `ReActAgent`를 통해 처리합니다. 내부 프롬프트는 대략 다음과 같은 구조를 가집니다:

```text
다음 질문에 최선을 다해 답하세요. 당신은 다음 도구들을 사용할 수 있습니다:
{도구_설명}

다음 포맷을 사용하세요:
질문: 당신이 답해야 하는 입력 질문
생각: 무엇을 해야 할지 항상 생각하세요
행동: 취해야 할 행동, [{도구_이름}] 중 하나여야 합니다
행동 입력: 행동에 대한 입력값
관찰: 행동에 대한 결과
... (생각/행동/행동 입력/관찰 단계가 N번 반복될 수 있습니다)
생각: 이제 최종 정답을 압니다
최종 답변: 원래 질문에 대한 최종 답변
```

### 요약

ReAct 패턴은 에이전트 세상의 '운영체제'와 같습니다. LLM이 실제 세상을 신뢰성 있게 탐색하는 데 필요한 구조적 규율을 제공하기 때문입니다. 모델에게 '행동하기 전에 먼저 생각하게' 함으로써, 우리는 AI를 단순한 텍스트 생성기에서 능동적인 문제 해결사로 진화시켰습니다.
